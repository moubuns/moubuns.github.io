Intruments Core Animation 使用简介

Core Animation 通过FPS来测量物理机上的图形性能
FPS: 一秒内屏幕刷新的次数
流畅的的屏幕刷新频率一般为60fps

视图的渲染分为两个部分CPU和GPU

CPU计算显示内容提交到GPU，GPU渲染完成后将渲染结果放入帧缓冲区，然后视频控制器把帧缓冲区的数据 传递给屏幕显示

CPU:视图的创建、布局计算、图片解码、文本绘制….
GPU:进行变换、合成、渲染….

模拟器和真机的差别

模拟器用CoreAnimaiton不能调试，在模拟器能显示 Offscreen-Rendered 等，但是不准确，手机设备的渲染和Mac存在一定的区别，推荐使用手机调试。

使用调节参数

  • Blended（混合） Layers
    绿色标记Layer不需要混合, 红色标记Layer需要叠加处理, GPU需要渲染多次来出，不同Layer的显示效果。
    如果食用带有alpha通道的图片，也会引起Blended Layers

优化：opaque 设置为ture 设置背景为不透明，尽量减少Clear Color的使用，但是…我们大部分切图本身就是带有Alpha通道的不透明切图…

减少设置背景透明，或者是这背景色，对中文无效，中文会生成一个opaque的subLayer

  • Hit Green and Misses Red

测试是否合适的使用栅格的缓存
栅格化:矢量图形转化为位图

第一次以进入的时候, 所有的东西都需要绘制所以都会标记为红色

  1. rasterize 仅适用于静态内容
  2. 栅格的缓存只有屏幕像素大小的2.5倍,6plus的像素为8M，也就是缓存的大小为20M
  3. 如果100ms内没有使用栅格图像的缓存则会从缓存中清除

栅格化的使用场景，避免重绘视图，如对一个动态模糊的图像进行上栅格化，减少动态模糊渲染的次数
优化：对于一些需要减少重复的视图设置layer.shouldRasterize = true

  • Copied Image
    青色的时候，图片的格式不能被GPU解析，需要使用CPU来进行解码

优化：对需要处理的图片的格式进行调整

  • Color Immedidately
    表示Instruments在做color-flush操作时取消10毫秒的延时，打开Hit Green and Misses Red会体验到使用效果

  • Misaligned Images
    对于比例缩放的图片标记为黄色，比如当你下载的图片时200 ＊ 200 实际显示的为100 ＊ 100
    对于放大的图像显示为青色

优化：需要避免缩放，在获取图片资源的时候，应该根据尺寸进行获取

  • Offscreen-Rendered Yellow

对于发生了离屏渲染的Layer标记为黄色，对于黄色标记的越多则发生的越多
离线渲染发生的场景比较多
如NaigationBar和TabBar需要离线模糊，shouldRasterize，DrawRect等都需要在渲染中增加CPU的处理负担

优化：圆角现实在CPU中计算，阴影的混合加入Path等

  • Compositing Fast-Path Blue
    蓝色的渲染层表示，使用硬件加速的时候，GPU会减少工作

  • Flash Updated Regions
    黄色表示频繁刷新的领域，越少刷新则CPU和GPU的消耗越小
